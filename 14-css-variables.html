CSS Variables (Custom Properties)

CSS variables, also known as custom properties, allow you to define reusable values that can be used throughout your stylesheets. This makes your CSS more maintainable, flexible, and easier to update.

1. Defining Variables

Variables are defined using two hyphens (`--`) followed by a name (e.g., `--primary-color`). They are usually defined within a pseudo-class like `:root` to make them globally accessible.

:root {
  --primary-color: #007bff;
  --main-bg-color: #f8f9fa;
}

2. Using Variables (`var()`)

To use a variable, you wrap its name with the `var()` function.

.button {
  background-color: var(--primary-color);
  color: white;
}

3. Scope of Variables

CSS variables are subject to the cascade and inheritance. They can be defined globally (on `:root`) or locally within specific selectors, limiting their scope.

Card Title

This card uses `--card-bg` and `--card-border` variables defined within its own scope.

.card {
  --card-bg: #e9ecef;
  background-color: var(--card-bg);
}

Dynamic Themings with Variables

Variables are incredibly powerful for implementing theme switching or dynamic styles.

Dark Theme Example

This section inherits and overrides some global variables to create a dark theme appearance.

.dark-theme {
  --main-bg-color: #343a40;
  --text-color: #f8f9fa;
  /* ... other overrides */
}

4. Fallback Values

You can provide a fallback value in the `var()` function. If the variable is not defined, the fallback value will be used.

This box uses a fallback color because `--non-existent-var` is not defined.

background-color: var(--non-existent-var, #ffeded);
